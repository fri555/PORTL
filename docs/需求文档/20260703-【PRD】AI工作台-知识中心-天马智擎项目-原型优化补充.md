## 二、P0 — 必须改

### 2.1 预览内容按格式差异化（当前最严重问题）

**现状**：所有文件格式共享同一套硬编码内容（B2B 线下团购方案），点开任何文件看到的内容一模一样。

**改为**：PDF / DOCX / MD / TXT 四种 MVP 格式各一套独立 Mock 内容，与文件类型和文件名强关联。

#### Mock 内容矩阵

| 格式 | 示例文件名 | 预览标题 | 文档大纲（导航目录） | 正文特色 |
|------|-----------|----------|---------------------|----------|
| **PDF** | 考勤管理制度_v3.pdf | 考勤管理制度 V3.0 | 总则 / 考勤方式 / 请假管理 / 加班管理 / 附则 | 条款式排版，少量表格（请假天数 vs 审批人） |
| **DOCX** | 员工手册2026版.docx | 员工手册 2026 版 | 第一章 总则 / 第二章 入职与试用 / 第三章 薪酬福利 / 第四章 行为规范 | 章节式排版，多级标题，正文段落较长 |
| **MD** | 运动鞋团购成功案例.md | 运动鞋团购成功案例 | 项目背景 / 客户需求 / 方案设计 / 执行复盘 / 关键经验 | Markdown 典型排版：列表、引用块、粗体强调 |
| **TXT** | 会议纪要_20260628.txt | 会议纪要 20260628 | 无大纲（纯文本不生成导航） | 纯文本展示，无格式，左上角提示"纯文本文件，暂不支持大纲导航" |

> **实现方式**：在 `allDocs` 中每个文档新增 `mockPreview` 字段，或为每个格式定义 mock 模板函数，`openPreview()` 时根据 `doc.format` 分派。

#### Mock 数据结构示意

```typescript
interface DocPreviewMock {
  title: string
  outline: { label: string; anchor: string }[]  // 导航目录
  sections: { anchor: string; content: string }[]  // 正文段落
  extraMeta?: { key: string; value: string }[]  // 额外元信息
}
```

### 2.2 恢复预览时自动打开小智问答

**现状**：`openPreview()` 中已移除 `qaOpen.value = true`，预览和小智解耦。

**改为**：恢复联动，但加一条规则——**如果用户手动关闭了小智，此次会话中不再自动弹出**。

```typescript
const qaManuallyClosed = ref(false)

function openPreview(doc: DocItem) {
  previewDoc.value = doc
  previewTabs.value = [doc, ...previewTabs.value.filter(item => item.name !== doc.name)]
  activeRightTab.value = doc.name
  // 联动打开小智（除非用户本次会话中手动关闭过）
  if (!qaManuallyClosed.value) {
    qaOpen.value = true
  }
}

function closeQaPanel() {
  qaOpen.value = false
  qaManuallyClosed.value = true  // 记录用户主动关闭
}
```

**理由**（用户确认的立场）：知识问答是知识中心的核心能力，看到文件应该自然联想到"我可以问关于这个文件的问题"。如果用户明确关闭了 QA，则尊重其意图，本会话不再自动打扰。

### 2.3 Mock 数据格式裁剪

**现状**：`allDocs` 中包含 XLSX（商品数据字典、预算执行表）、PPTX（AI项目说明等），超出 MVP 格式范围。

**改为**：
- 保留格式：**PDF、DOCX、MD、TXT**
- 移除格式：XLSX、PPTX（标记为 "后续版本 ×" 或在 UI 中展示灰色禁用态）
- 受影响的文件标记为"该格式暂不支持在线预览，后续版本支持"

如果不希望移除现有 mock 数据，可在 `openPreview()` 中加格式拦截：

```typescript
const MVP_PREVIEW_FORMATS = ['PDF', 'DOCX', 'MD', 'TXT']
function openPreview(doc: DocItem) {
  if (!MVP_PREVIEW_FORMATS.includes(doc.format)) {
    showFileActionToast(`${doc.format} 格式暂不支持在线预览，后续版本支持`)
    return
  }
  // ... 正常预览逻辑
}
```

## 三、P1 — 应该改

### 3.1 文档大纲动态化

**现状**：左侧 132px 导航栏固定展示「方案摘要 / 预算分档 / 执行建议」三个条目，与实际打开的文件无关。

**改为**：根据 `doc.format` 和 `doc.mockPreview.outline` 动态渲染导航列表。纯 TXT 文件提示"纯文本文件无大纲"。

效果：

| 格式 | 导航条目 | 无导航时的表现 |
|------|---------|---------------|
| PDF | 总则 / 考勤方式 / 请假管理 / 加班管理 / 附则 | — |
| DOCX | 第一章 总则 / 第二章 入职与试用 / 第三章 薪酬福利 / 第四章 行为规范 | — |
| MD | 项目背景 / 客户需求 / 方案设计 / 执行复盘 / 关键经验 | — |
| TXT | — | 左上角灰底提示"纯文本文件，暂不支持大纲导航" |

### 3.2 引用高亮增强

**现状**：点击 `[[ref:1]]` 后章节标题出现短暂高亮动画，用户滚动后消失；且高亮只定位到章节标题，没有定位到具体的正文段落。

**改为**：

1. **滚动定位**：点击引用后，正文区域自动滚动到对应章节（使用 `scrollIntoView({ behavior: 'smooth', block: 'center' })`）
2. **持久高亮**：被引用的段落添加 `highlighted-paragraph` CSS 类（持续高亮背景色，直到用户点击另一个引用或主动关闭高亮）
3. **参考锚点映射**：`getDocSection()` 扩展为 `getDocAnchor(docName, refIndex)`，返回具体的段落锚点而非固定的章节标题名

```typescript
function openCitationRef(index: number, citations?: string[]) {
  const docName = citations?.[index - 1]
  if (!docName) return
  highlightedSection.value = null
  // 1. 打开文档（复用已有逻辑）
  for (const kbId of Object.keys(allDocs)) {
    const doc = allDocs[kbId].find((d: DocItem) => d.name === docName)
    if (doc) { openPreview(doc); break }
  }
  // 2. 滚动到目标段落（下一 tick 确保 DOM 已渲染）
  nextTick(() => {
    const anchor = getDocAnchor(docName, index)
    const el = document.getElementById(`preview-para-${anchor}`)
    el?.scrollIntoView({ behavior: 'smooth', block: 'center' })
    highlightedSection.value = anchor  // 持久 class
  })
}
```

### 3.3 预览顶部元信息卡片

**现状**：预览面板顶部只展示「修改于 X · Y」。大量用户关心的信息未展示。

**改为**：预览面板顶部（标题下方）增加一行元信息卡片：

```
┌─────────────────────────────────────────┐
│ 考勤管理制度_v3.pdf                     │
│ ┌──────┐ ┌──────────┐ ┌──────────────┐ │
│ │ PDF  │ │ 🔒 内部   │ │ 📎 制度 · 行政 │ │ ← 格式 + 密级 + 标签
│ └──────┘ └──────────┘ └──────────────┘ │
│ 王管理员 · 2026-06-15 · 2.4 MB         │ ← 上传人 + 时间 + 大小
└─────────────────────────────────────────┘
```

展示的字段（从现有 `DocItem` 和 `allDocs` 获取）：

| 字段 | 来源 | 当前状态 |
|------|------|---------|
| 格式徽标 | `doc.format` | 已有（getIconColor） |
| 安全等级 | `doc.securityLevel` | 已有（getDocSecurityClass） |
| 标签 | `doc.tags` | 已有 |
| 上传人 | `doc.uploadedBy` | 已有（当前只在下方小字显示） |
| 更新时间 | `doc.updatedAt` | 已有 |
| 文件大小 | `doc.size` | 已有定义但未展示 |
| **状态**（已索引/解析中/待审核） | `doc.status` | 已有但未在预览面板展示 |

## 四、P2 — 值得改

### 4.1 QA 上下文栏（快捷引用知识库/文件）

**核心思路**：用户能在小智面板中看到"当前问答的范围是什么"，并能快速切换/增删范围，而不必离开当前页面去文件树导航。

#### 设计方案

在小智面板顶部输入框上方增加一条「上下文栏」：

```
┌─────────────────────────────────────┐
│ 小智 · 知识库问答助手               │
│                                     │
│ ┌─────────────────────────────────┐ │
│ │ 📚 方案中心案例库           ✕  │ │ ← 当前搜索范围：知识库
│ │ 📄 运动鞋团购成功案例.md    ✕  │ │ ← 当前搜索范围：具体文件
│ │ [+ 添加知识库] [+ 添加文件]    │ │ ← 快速切换/增删
│ └─────────────────────────────────┘ │
│                                     │
│ [问答消息记录区域...]               │
│                                     │
│ ┌─────────────────────────────────┐ │
│ │ 输入框...               [↑]    │ │
│ └─────────────────────────────────┘ │
└─────────────────────────────────────┘
```

**默认行为**：
- 用户通过点击文件进入预览 → KB + 该文件自动加入上下文
- 用户通过点击 KB 进入 → 该 KB 自动加入上下文
- 用户手动打开小智（未选任何 KB/文件）→ 提示"请选择知识库或打开文件后开始提问"

**交互规则**：
- 上下文栏支持拖入/移出 KB 或文件
- 点击 `✕` 从上下文中移除某个源（不影响页面当前选中状态）
- 点击 `[+ 添加知识库]` → 弹出轻量级选择器（搜索或快速切换）
- 点击 `[+ 添加文件]` → 弹出当前 KB 内文件选择器
- 上下文变更后，下一条问答重新检索

#### 为什么不做 @"提及" 语法

@ 语法需要在输入框中运行 mention parser，复杂度明显上升（需处理光标位置、模糊匹配、退格删除等边界），但增量价值有限——用户更习惯点击选择而非记忆并输入名字。P2 阶段用「上下文栏 + 选择器弹窗」的方式更轻量、更直观。

### 4.2 预览面板宽度可拖拽

**现状**：`previewPanelWidth = 'clamp(390px,32vw,520px)'`，用户无法调整。

**改为**：在预览面板左侧添加可拖拽分隔条。

```html
<div
  class="absolute left-0 top-0 h-full w-1 cursor-col-resize hover:bg-blue-300 active:bg-blue-500"
  @mousedown="startResize"
/>
```

```typescript
function startResize(e: MouseEvent) {
  const startX = e.clientX
  const startW = previewWidthPx.value
  const handler = (ev: MouseEvent) => {
    previewWidthPx.value = Math.max(320, Math.min(800, startW + ev.clientX - startX))
  }
  document.addEventListener('mousemove', handler)
  document.addEventListener('mouseup', () => document.removeEventListener('mousemove', handler), { once: true })
}
```

### 4.3 预览面板内全文搜索

在预览正文区域上方添加一个搜索条（Ctrl+F 触发）：

```
┌─────────────────────────────────────┐
│ [🔍 搜索文档内容...]   [3/12] [⬆][⬇][✕] │
├─────────────────────────────────────┤
│ ...正文内容，匹配关键字高亮...      │
└─────────────────────────────────────┘
```

MVP 阶段只需前端纯文本搜索 + 高亮，不需要后端接口。

### 4.4 加载状态（骨架屏）

`openPreview()` 打开时，先用骨架屏占位 300-500ms（模拟接口响应时间），再填充内容。

```html
<template v-if="previewLoading">
  <div class="animate-pulse space-y-4 p-6">
    <div class="h-6 w-1/3 rounded bg-zinc-200" />
    <div class="h-4 w-2/3 rounded bg-zinc-100" />
    <div class="h-4 w-full rounded bg-zinc-100" />
    <div class="h-4 w-5/6 rounded bg-zinc-100" />
  </div>
</template>
```

## 六、交互流程规范（整合版）

### 6.1 点击文件 → 预览

```
用户点击文件
  → openPreview(doc)
  → 格式校验（仅支持 PDF/DOCX/MD/TXT，否则 toast 提示）
  → 预览面板打开（中间内容区替换为预览视图）
  → 小智问答自动打开（除非 qaManuallyClosed）
  → 上下文栏自动添加上下文：当前 KB + 当前文件
  → 骨架屏 → 300ms 后展示 Mock 内容
```

### 6.2 点击引用 → 定位

```
用户在小智回答中点击 [[ref:1]]
  → openCitationRef(index, citations)
  → openPreview(doc)（如果尚未打开）
  → 高亮清除 previous
  → scrollIntoView 定位到目标段落
  → highlighted-paragraph CSS 类持久高亮（amber 背景）
```

### 6.3 手动开关面板

```
用户点击「折叠预览」→ previewTabs = [] → 回到文件列表
用户点击「关闭小智」→ qaOpen = false + qaManuallyClosed = true
用户点击「打开小智」（通过头部的"知识库问答"按钮）→ qaOpen = true + qaManuallyClosed = false
用户切换 Tab → activeRightTab 切换 → 大纲和正文章节随之刷新
```
