## 2. 业务背景与目标

### 2.1 天马集团业务背景

南京天马网络科技集团是运动鞋服赛道的复合型企业，具备三重身份：

1. **运动鞋服电商品牌方**：运营自有/代理运动鞋服品牌，覆盖天猫、京东、抖音、得物、拼多多等多平台，在营 SKU 数千个。
2. **运动鞋服供应链/产业带平台企业**：连接上游工厂与下游中小商家，提供货源、趋势、选品服务。
3. **自营云仓**：自有仓储履约体系，掌握真实库存、动销、周转数据。

集团具备成熟的产研团队（10+ 人），已有 ERP/OMS/云仓系统，具备自建爬虫、AI 调用、前后端开发能力。

### 2.2 核心痛点

| 痛点 | 现状影响 | 量化参考（竞品报告） |
|------|---------|---------------------|
| 商拍成本高、周期长 | 传统商拍 ¥1000-3500/款，单款周期 1-2 周 | 知衣 FD+ 将单件商拍降至约 ¥40 |
| 内容生产跟不上上新速度 | 数千 SKU × 多平台，人工出图/视频严重不足 | 设计周期从 2 周→3 天 |
| 爆品判断靠经验 | 选品录用率约 30%，备货易错 | 知衣爆款预测将录用率提至 70% |
| 市场数据分散 | 竞品/趋势/价格/评论散落各平台，无统一数据底座 | 蝉妈妈分钟级数据延迟低于行业 50% |
| 内容效果无法闭环 | 生成内容投放后效果数据回不来，无法优化 | Kickart 转化数据回传→AI 优化飞轮 |
| 云仓库存与内容脱节 | 畅销款内容供给不足、滞销款内容浪费 | 缺乏库存×内容联动决策 |

### 2.3 产品目标

**总目标**：构建「数据采集 → 爆品预测 → 内容生成 → 决策应用」全链路 AI 内容工厂，以运动鞋服垂直化能力，将天马的内容生产成本降低 80%+、爆款命中率提升 50%+，并形成可对平台下游商家输出的标准化能力。

**一期（MVP，内部）量化目标**：

| 指标 | 目标值 | 基线参考 |
|------|--------|---------|
| 商拍单件成本 | ≤ ¥60 | 传统 ¥1000-3500 |
| 商品图出图效率 | ≥ 30 张/分钟/人 | 人工约 1 张/小时 |
| 营销短视频产出 | ≥ 50 条/天/运营 | 人工约 3-5 条/天 |
| 详情页素材齐全率 | ≥ 95% SKU | 现状约 60% |
| 爆品预测准确率 | ≥ 65%（7 日起爆） | 经验选品约 30% |
| 内容投放效果回灌率 | ≥ 80% | 现状 0% |

**二期（开放平台商家）目标**：将内容工厂能力 SaaS 化，多租户隔离，服务云仓/产业带平台下游商家，形成第二增长曲线。

## 3. 产品规划

### 3.1 产品定位

> **天马运动鞋服 AI 内容工厂**：以运动鞋服垂直 AI 内容生成为核心，打通「市场数据采集 → 爆品/趋势预测 → 多场景内容生成 → 投放决策与效果闭环」的运营操作系统，先服务天马内部提效，后开放给平台下游商家。

一句话定位：**让运动鞋服企业拥有一个 7×24 小时的 AI 内容与运营部门。**

### 3.2 目标用户（一期内部）

| 角色 | 核心诉求 | 使用模块 |
|------|---------|---------|
| 品牌运营经理 | 全局看板、决策支持 | 决策应用、爆品预测 |
| 选品经理 | 爆品识别、趋势把握 | 数据采集、爆品预测 |
| 内容设计师 | 高效出图/视频、降低重复劳动 | 内容生成（4 场景） |
| 投放运营 | 素材批量生产、效果追踪 | 内容生成、决策应用 |
| 电商店长 | 多平台内容适配、上架效率 | 内容生成、决策应用 |
| 供应链/云仓管理 | 库存×内容联动、备货决策 | 决策应用（库存联动） |

### 3.3 价值主张

| 对象 | 价值主张 |
|------|---------|
| 内部运营团队 | 内容生产降本 80%+，爆款命中率 +50%，多平台一键适配 |
| 集团管理层 | 数据资产沉淀，决策从经验驱动到数据+AI 驱动 |
| 云仓/供应链 | 库存与内容联动，畅销款内容加码、滞销款止损 |
| 二期平台商家 | 获得头部品牌同款 AI 内容能力，降本增效 |

### 3.4 分期规划

```
一期（MVP · 内部 · 3 个月）           二期（开放平台 · +3 个月）
├─ 数据采集（轻量版）                 ├─ 多租户改造
├─ 爆品预测（MVP 版）                 ├─ 商家自助工作台
├─ 内容生成（4 场景，核心）           ├─ 能力计费/套餐
├─ 决策应用（效果看板）               ├─ API 开放
└─ 与 ERP/OMS/云仓集成               └─ 平台商家数据隔离
```

### 3.5 不做什么（YAGNI）

- **不自训基础大模型**：生图/视频/LLM 一律调用外部 API + 自研垂直工作流，不重复造轮子。
- **一期不做 3D 仿真/打版**：参考知衣短板，3D 非本期范围，必要时与 Style3D/CLO 3D 集成。
- **一期不做出海多语言**：聚焦国内运动鞋服市场。
- **一期不做实时直播数字人**：聚焦内容素材生产，直播数字人留二期评估。

## 5. 产品总体架构

### 5.1 四层流水线架构

```
┌─────────────────────────────────────────────────────────────────┐
│                      决策应用层（Decision）                        │
│   效果看板 · 投放渠道建议 · 选品决策 · 库存×内容联动 · AI 周报     │
└───────────────────────────┬─────────────────────────────────────┘
                            │ 效果回灌
┌───────────────────────────┴─────────────────────────────────────┐
│                    内容生成层（Content）★核心                      │
│  AI商拍/模特上身 · 商品图/详情页 · 营销短视频 · 文案脚本            │
│  ＋ 品牌资产库 · 鞋服标签体系 · 提示词模板库 · 质检                │
└───────────────────────────┬─────────────────────────────────────┘
                            │ 爆品/趋势输入
┌───────────────────────────┴─────────────────────────────────────┐
│                   分析预测层（Analysis）                          │
│  爆品预测ML · 趋势分析 · 价格带分析 · 消费者洞察 · 竞品监控        │
└───────────────────────────┬─────────────────────────────────────┘
                            │ 原始数据
┌───────────────────────────┴─────────────────────────────────────┐
│                    数据采集层（Collection）                       │
│  运动鞋服专项爬虫（天猫/京东/抖音/得物 + 品牌官网）· 数据清洗中台  │
└─────────────────────────────────────────────────────────────────┘
        │                                          ↑
        └──── 数据左→右流动 ────→ 内容生成 ────→ 决策应用 ──┘
```

**数据流说明**：
1. 数据采集层抓取运动鞋服市场数据（竞品商品/价格/销量/评论/趋势）。
2. 分析预测层基于采集数据 + 云仓内部数据，输出爆品预测、趋势、价格带洞察。
3. 内容生成层接收"市场洞察 + 商品素材 + 品牌资产"，按 4 场景生成内容。
4. 决策应用层将内容分发到各平台，追踪效果，并将效果数据回灌分析预测层，形成飞轮。

### 5.2 与现有系统集成

| 现有系统 | 集成方式 | 数据交互 |
|---------|---------|---------|
| ERP | API 双向 | 商品 SKU/属性/库存 → 内容生成；内容素材 → ERP 素材库 |
| OMS | API 只读 | 订单/动销数据 → 爆品预测、决策应用 |
| 云仓 | API 双向 | 库存/周转 → 库存×内容联动决策；内容需求 → 备货建议 |
| 电商店铺（天猫/京东/抖音/得物） | 开放平台 API | 内容分发上传、效果数据回采 |

## 7. 详细功能需求

### 7.1 数据采集模块

#### 7.1.1 功能概述

构建运动鞋服专项爬虫矩阵，采集竞品商品、价格、销量、评论、趋势数据，为爆品预测和内容生成提供输入。采用"自建为主、Apify/Bright Data 为辅"策略。

#### 7.1.2 功能清单

| 编号 | 功能 | 说明 | 优先级 |
|------|------|------|--------|
| F-DC-01 | 平台商品采集 | 采集天猫/京东/抖音/得物运动鞋服类目商品（标题/价格/销量/图片/属性） | P0 |
| F-DC-02 | 品牌官网采集 | 采集 Nike/Adidas/安踏/李宁官网商品（SKU/配色/尺码/科技属性） | P0 |
| F-DC-03 | 评论采集 | 采集商品评论（评分/文本/尺码体验/舒适度，参考 Nike Reviews Scraper 字段） | P1 |
| F-DC-04 | 价格监控 | 定时追踪竞品价格变化，异常降价告警 | P1 |
| F-DC-05 | 数据清洗中台 | 多源数据统一清洗、去重、标准化、入库 | P0 |
| F-DC-06 | 反爬管理 | 代理池管理、请求频率控制、失败重试、UA 轮换 | P0 |
| F-DC-07 | 采集任务调度 | 可配置定时任务、并发控制、失败告警 | P0 |
| F-DC-08 | 数据质量校验 | 字段完整性、异常值检测、数据新鲜度监控 | P1 |

#### 7.1.3 采集字段标准（运动鞋服专项）

```
商品基础：商品ID/标题/链接/品牌/品类/子品类/性别
定价：当前价/原价/折扣率/价格带
规格：配色/尺码列表/SKU变体/库存状态
属性（鞋服专项）：
  鞋类 → 鞋型/鞋面材质(网布/皮革/飞织)/鞋底科技(气垫/碳板)/重量/跟尖差
  服装 → 版型(紧身/宽松)/面料(速干/透气/弹力)/功能/适用运动
媒体：图片URL列表/视频URL
反馈：评分/评论数/评论摘要/尺码体验/舒适度
趋势：销量趋势/搜索热度/上架时间
```

#### 7.1.4 技术实现要点

- 自建爬虫框架：Crawlee（Apify 开源）或 Scrapy，Python/Node 双栈。
- 代理：住宅代理为主（应对 Nike/Adidas 反爬），数据中心代理兜底。
- 参考 Apify 分层模式：列表级（轻量、低成本）→ 详情级（深度 SKU/尺码）→ 评论级。
- 数据清洗：统一 schema → 去重 → 标准化品类/属性 → 入数据仓库。
- 频率：商品/价格每日 1-2 次；销量趋势每日；评论每周。

#### 7.1.5 合规要求

- 遵守目标平台 ToS，控制频率避免影响对方服务。
- 不采集个人隐私数据，评论脱敏处理。
- 商业化前评估法律合规（参考 Apify 报告风险提示）。

### 7.2 爆品分析预测模块

#### 7.2.1 功能概述

基于采集的市场数据 + 云仓内部动销数据，构建爆品预测 ML 模型与趋势/价格带分析，为选品决策和内容生成提供"生成什么、给谁生成"的输入。

#### 7.2.2 功能清单

| 编号 | 功能 | 说明 | 优先级 |
|------|------|------|--------|

#### 7.2.3 爆品预测模型（MVP）实现逻辑

```
特征工程：
├─ 商品属性特征：品类/价格/品牌/功能标签/配色/上架时间
├─ 市场特征：搜索热度趋势/同类品销量增速/价格带竞争度
├─ 内容特征：主图数量/视频有无/详情页完整度
├─ 时序特征：上架后 N 日销量曲线/增长率/加速度
└─ 云仓特征：同类品历史动销/库存周转/退货率（天马独有）

模型：LightGBM/XGBoost 二分类（起爆 vs 不起爆）+ 回归（预测销量区间）
训练数据：历史采集数据 + 云仓动销数据（标注）
评估指标：准确率(P0≥65%)、召回率、AUC
迭代：每周用新数据重训，预测效果回灌（F-AP-07）
```

#### 7.2.4 输出给内容生成层的接口

爆品预测模块向内容生成层输出"内容生成任务建议"：
- 高起爆概率 SKU → 优先排期生成全套内容（商拍+商品图+视频+文案）。
- 趋势品类/功能/配色 → 指导内容生成的卖点重点和场景选择。
- 价格带洞察 → 指导内容中的价值传达策略。

### 9.4 内容生成编排逻辑（核心）

```python
# 伪代码：内容生成 Agent 编排逻辑
def content_generation_agent(task):
    # 1. 任务解析
    sku = task.sku  # 来自 ERP
    scenario = task.scenario  # 商拍/商品图/视频/文案
    market_insight = predict_service.get_insight(sku)  # 来自预测层
    brand_assets = asset_library.get(task.brand_id)

    # 2. 商品理解 + 标签匹配
    tags = product_understanding(sku, sku.images)
    workflow = select_workflow(scenario, tags)  # 选工作流(服装VTON/鞋类角度/...)

    # 3. 工作流执行
    if scenario == "商拍":
        results = run_commercial_shoot_workflow(
            sku, tags, brand_assets, market_insight
        )
    elif scenario == "商品图":
        results = run_product_image_workflow(...)
    elif scenario == "视频":
        results = run_video_workflow(...)
    elif scenario == "文案":
        results = run_copywriting_workflow(...)

    # 4. 质检
    qc_results = auto_qc(results, brand_assets)
    if qc_results.has_issues:
        results = auto_fix(results, qc_results)

    # 5. 人工复核（关键素材）
    if task.is_critical:
        await human_review(results)

    # 6. 入库 + 分发
    asset_library.save(results)
    distribution_service.dispatch(results, task.platforms)

    # 7. 效果回灌注册
    tracking_service.register(results, task)
```

### 9.5 数据流与效果回灌闭环

```
采集层 →(原始数据)→ 预测层 →(爆品/趋势)→ 生成层 →(内容)→ 决策层
   ↑                                                        │
   └────────────(效果数据回灌)←──────────────────────────────┘
```

效果回灌实现：
1. 内容分发后，决策层通过电商开放平台 API 采集曝光/点击/转化/GMV。
2. 效果数据关联到内容 ID、SKU、场景、平台。
3. 周期性回灌：效果数据 → 预测层（优化爆品模型）+ 生成层（优化模板/提示词）。
