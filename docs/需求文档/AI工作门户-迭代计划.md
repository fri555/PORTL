# AI工作门户 迭代计划

> 版本：v2.0  
> 日期：2026-06-25  
> 适用范围：AI中台第一期 MVP 及后续迭代  
> 核心原则：第一版只保留首页搜索入口和全屏对话工作台，先验证种子用户的方案中心场景；需求池、能力市场、工作台首页后置。

## 1. 迭代总览

| 阶段 | 周期 | 目标 | 主要交付 |
|------|------|------|----------|
| Phase 0 | 第 0 周 | 上线准备 | 种子用户、方案中心知识、权限账号、首发能力确认 |
| Phase 1 | 第 1-3 周 | 一期 MVP | 首页搜索、全屏对话工作台、方案中心主管、基础权限 |
| Phase 2 | 第 4-6 周 | 方案中心增强 | 文件上传、案例池、字段池、SKU/知识检索、钉钉能力联调 |
| Phase 3 | 第 7-10 周 | 运营与闭环 | 需求池、AIBP协同、后台用量、通知提醒 |
| Phase 4 | 第 11 周后 | 平台化演进 | 能力市场、MCP市场、创建智能体、定时任务、数据看板 |

## 2. Phase 0：上线准备

| 事项 | 责任人 | 输出物 |
|------|--------|--------|
| 种子用户确认 | 项目负责人 + 平台部 | B2B线上、B2B线下、服务商、门店等名单 |
| 方案中心主管定义 | AI项目组 + 业务负责人 | 智能体目标、提示词、能力边界 |
| 首发快捷功能确认 | AI项目组 | 通用问答、方案生成、营销文案、文档总结、PPT提纲、钉钉能力清单 |
| 权限账号准备 | 系统管理员 | 普通用户、种子用户、运营管理员、系统管理员 |
| 方案知识准备 | 方案中心负责人 | 通用方案模板、成功案例、字段清单、商品知识接入计划 |
| 技术联调准备 | 后端/集成负责人 | 钉钉登录、钉钉待办、钉钉文档、钉钉AI表格接口状态 |

### 准入标准

- 种子用户范围已确认
- 方案中心主管提示词和首批模板已确定
- 首页可用快捷功能清单已冻结
- 基础权限模型和管理员账号已可用
- 钉钉能力明确“可直连 / 可跳转 / 先占位”的状态

## 3. Phase 1：一期 MVP

### 第 1 周：首页与权限

| 功能 | 交付内容 | 验收 |
|------|----------|------|
| 首页搜索框 | 支持输入问题、任务和方案需求 | 输入后可进入全屏工作台 |
| 快捷功能 | 只保留基础通用能力和方案中心入口 | 无能力市场、无需求入口 |
| 普通用户视图 | 普通用户只看到首页和授权能力 | 不展示后台、管理、需求池 |
| 权限拦截 | 未授权能力不可见或不可调用 | 有明确拦截提示和日志 |

### 第 2 周：全屏对话工作台

| 功能 | 交付内容 | 验收 |
|------|----------|------|
| 主对话区 | 多轮问答、复制结果、继续追问 | 能完成连续方案沟通 |
| 历史会话 | 左侧最近会话与切换 | 能找回历史方案 |
| 能力上下文 | 右侧展示智能体、知识库、工具状态 | 用户知道调用了什么能力 |
| 工具调用状态 | 展示 MCP/钉钉能力调用中、成功、失败 | 异常有说明 |

### 第 3 周：方案中心主管与联调

| 功能 | 交付内容 | 验收 |
|------|----------|------|
| 方案中心主管 | 需求澄清、方案初稿、方案优化 | 能在 5 分钟内输出一版可讨论方案 |
| 营销文案撰写 | 输出客户沟通话术、卖点文案 | 可复制可二次编辑 |
| 钉钉待办 | 对话中生成待办并同步/占位 | 明确联调或降级状态 |
| 钉钉文档 | 对话结果整理为文档草稿/跳转 | 文档标题和正文结构可用 |
| 钉钉AI表格 | 将字段、清单、方案要素转为表格结构 | 表头和行数据可用 |

## 4. Phase 2：方案中心增强

| 能力 | 说明 |
|------|------|
| 文件上传 | 支持 Excel、Doc、PDF 等资料上传，限制大小和类型 |
| 在线预览定位 | 支持引用资料在线预览和定位来源 |
| 成功案例池 | 按客户类型、行业、价位、类目沉淀案例 |
| 字段池 | 通用方案、个性化方案、成功案例使用不同字段池 |
| SKU检索 | 接入 10W+ SKU，可用于组货建议 |
| PPT生成MCP | 将方案初稿转为 PPT 结构或文件 |

## 5. Phase 3：运营与闭环

| 能力 | 说明 |
|------|------|
| 需求池 | 上线需求提报、状态流转、负责人、时间线 |
| AIBP协同 | 支持业务需求收集、价值判断、跟进复盘 |
| 管理后台增强 | 查看部门使用、会话量、工具调用、权限配置 |
| 通知提醒 | 需求状态、任务待办、能力上线通过钉钉触达 |
| 试点复盘 | 输出种子用户使用数据、成功案例和下一轮需求 |

## 6. Phase 4：平台化演进

| 能力 | 说明 |
|------|------|
| 工作台首页 | 建立完整智能工作台入口、资产概览和创建入口 |
| 能力市场 | 智能体、MCP、技能、专家团市场 |
| 创建智能体 | 面向管理员和开发者开放配置能力 |
| 创建知识库 | 支持部门自建、审核、权限配置 |
| 定时任务 | 触发器、日历、周期任务和自动执行 |
| 数据看板 | AI使用、成本、效率、需求漏斗分析 |

## 7. 优先级规则

### P0：第一版必须完成

- 首页搜索框
- 首页基础快捷功能
- 全屏对话工作台
- 方案中心主管
- 基础权限管理
- 普通用户隐藏后台
- 会话和工具调用审计日志

### P1：第一版后优先补

- 文件上传
- 成功案例池
- 钉钉待办/文档/AI表格深度联调
- SKU 检索
- PPT 生成 MCP

### P2：稳定后再做

- 需求池
- 工作台首页
- 能力市场
- 创建智能体/知识库/技能
- 定时任务
- 数据看板

## 8. 迭代判断标准

每个需求进入迭代前先回答四个问题：

1. 是否直接帮助种子用户更快完成方案沟通？
2. 是否能缩短“需求澄清 - 方案初稿 - 修改交付”的时间？
3. 是否能复用方案中心知识、案例、SKU 或钉钉协同能力？
4. 是否会让第一版入口变复杂？

如果第 4 个问题答案是“会”，默认后置。
