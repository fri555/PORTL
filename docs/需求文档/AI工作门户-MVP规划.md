# AI工作门户 一期MVP规划

> 版本：v2.0  
> 日期：2026-06-25  
> 适用范围：AI中台第一期 MVP / 种子用户试点  
> 核心结论：第一版继续做减法，只保留“首页搜索框 + 文件上传 + 全屏对话工作台 + 方案中心主管智能体 + 子智能体-组货方案专家 + 必要知识库/MCP/触发器/Skill管理入口 + 基础权限管理”。工作台首页、能力市场、需求提报、需求池、复杂智能体创建均不进入第一版。

## 1. MVP 定位

第一期 MVP 的目标不是做完整 AI 门户，而是给种子用户提供一个可真实使用的 AI 中台入口：

1. 用户从首页输入问题或任务。
2. 用户可上传 Excel、Word、PDF、PPT、图片等资料作为上下文。
3. 系统根据快捷功能、默认能力或指定智能体进入全屏对话工作台。
4. 对话工作台承接多轮问答、历史会话、知识库/MCP 工具调用和结果输出。
5. 管理侧具备基础权限控制、用户角色、智能体/工具可见性配置。

普通用户界面不强调“第一期”“MVP”“试点”等内部项目状态，只呈现为正式可用的 AI 工作入口。

完整用户旅程和长期功能地图见：`AI工作门户-用户故事地图.xlsx`。本文件只定义第一期可落地范围。

## 2. 种子用户与首发场景

基于与平台部“天团一号”沟通结果，第一期种子用户围绕“方案中心”场景展开。

| 种子用户/对象 | 典型场景 | 关键诉求 |
|----------------|----------|----------|
| B2B线上团队 | 企业微信沟通、线上客户方案回复 | 沟通窗口短，需要快速澄清需求并输出方案 |
| B2B线下团队 | 到店沟通、门店/服务商现场方案 | 沟通时间更短，需要 5 分钟内出一版可讨论方案 |
| 内部服务商 | 支撑集团内部方案、组货、报价、文档输出 | 需要复用成功案例、通用模板和商品知识 |
| 外部服务商 | 面向外部客户做方案沟通与交付 | 需要标准化方案结构和可控输出口径 |
| 线下零售门店 | 门店客户需求收集、方案初稿生成 | 需要低门槛输入、快速得到可转述内容 |

## 3. 第一版只做什么

### 3.1 首页

首页是第一版唯一前台入口。

| 模块 | MVP 内容 | 不做内容 |
|------|----------|----------|
| 搜索/任务输入框 | 支持输入咨询问题、方案需求、文案任务；支持附件上传；发送后跳转全屏对话工作台 | 不做自由桌面组件、不做资讯流、不做需求入口 |
| 快捷功能 | 只保留基础通用能力和已可接入能力；每个快捷功能必须有对应案例 | 不放复杂市场分类、不放实验型能力 |
| 今日/推荐示例 | 围绕方案中心、通用办公、营销文案、钉钉协同展示少量示例；切换快捷功能时案例同步变化 | 不做大面积推荐流 |
| 移动端首页 | 只保留搜索框和快捷功能入口 | 不展示复杂卡片和管理入口 |

### 3.2 全屏对话工作台

用户从首页搜索框发起任务后，进入全屏对话工作台。

| 模块 | MVP 内容 | 验收标准 |
|------|----------|----------|
| 主对话区 | 支持文本输入、附件上下文、多轮问答、结果复制 | 用户能围绕一个方案任务连续对话 |
| 历史会话 | 左侧展示历史会话，支持切换最近会话 | 用户能找到刚才的方案或任务 |
| 能力上下文 | 右侧展示当前智能体、引用文件、输出文件、知识缺口、知识库和MCP工具状态 | 用户知道当前对话调用了什么能力、引用了哪些资料、生成了哪些交付物 |
| 知识库上传 | 左侧提供上传到知识库入口，支持组货逻辑、预算池、成功案例、字段定义、PPT/PDF模板入库 | 方案中心运营可持续补充专家依赖资料 |
| 方案中心主管 | 首发唯一特殊智能体，负责标准字段采集、需求澄清、方案池匹配、案例引用、商品组货建议 | 能完成一轮“需求澄清 - 方案初稿 - 优化建议” |
| 子智能体-组货方案专家 | 承接方案中心主管下发的组货任务，负责预算分档、品牌偏好、方案池匹配、SPU/SKU待查表、知识缺口沉淀 | 能输出三档组货方案、内部待查字段和待补知识项 |
| 能力管理入口 | 左侧保留知识库、MCP连接器、触发器、Skill编写管理入口 | 用户能知道当前任务依赖哪些底层能力 |
| 通用工具调用 | 支持已接入 MCP/连接器能力的占位与调用状态展示 | 工具调用有明确成功/失败反馈 |

### 3.3 基础权限管理

权限管理是后台基础能力，普通用户不展示。

| 对象 | 权限口径 |
|------|----------|
| 普通用户 | 只能看到首页、授权快捷功能、全屏对话工作台 |
| 种子用户 | 在普通用户基础上可使用“方案中心主管”智能体和相关知识库 |
| 方案中心主管/运营管理员 | 可维护方案中心提示词、知识库、案例池和可用工具 |
| 系统管理员 | 可配置用户、部门、角色、菜单、智能体、知识库、MCP 工具权限 |

## 4. 首发能力清单

### 4.1 首页快捷功能

第一版只放基础、通用、公开已有成熟范式的能力，不追求覆盖全部 AI 模式。

| 快捷功能 | 目标用户 | 说明 |
|----------|----------|------|
| 通用问答 | 全体种子用户 | 通用咨询、总结、解释、改写 |
| 方案生成 | 方案中心种子用户 | 进入“方案中心主管”，用于需求澄清和方案初稿 |
| 营销文案撰写 | 市场/方案/运营人员 | 商品卖点、客户沟通话术、活动文案 |
| 文档总结 | 全体种子用户 | 总结文档、会议材料、客户需求描述 |
| PPT提纲 | 方案/管理人员 | 生成方案汇报结构，不在第一版生成完整 PPT |
| 钉钉待办 | 需要协同跟进的用户 | 将待办事项同步或生成到钉钉待办，第一版可先做接口占位/联调 |
| 钉钉文档 | 需要沉淀文档的用户 | 将对话结果整理为钉钉文档，第一版可先做跳转或草稿能力 |
| 钉钉AI表格 | 需要结构化整理的用户 | 将清单、方案字段、客户需求转为表格结构，第一版可先做草稿导出 |

### 4.2 首发特殊智能体

| 智能体 | 是否进入第一版 | 说明 |
|--------|----------------|------|
| 方案中心主管 | 是 | 唯一首发特殊智能体 |
| 子智能体-组货方案专家 | 是 | 方案中心主管下的核心子智能体，用于组货方案生成 |
| AI办公助理 | 否 | 暂由通用问答和快捷功能覆盖 |
| 电商运营专家 | 否 | 后续根据种子用户效果再拆分 |
| AIBP需求助理 | 否 | 第一版不做需求池 |
| 数字人直播智能体 | 否 | 不进入第一期 |

### 4.3 方案中心主管能力边界

| 能力 | MVP 做法 | 后续扩展 |
|------|----------|----------|
| 标准字段采集 | 以表单提示方式收集客户类型、数量、预算、活动场景、品牌偏好、颜色与男女比例、交付时间等字段 | 字段校验、字段自动补全、字段版本管理 |
| 需求澄清 | 用标准话术追问模糊或不真实需求，先确认真实用途和决策偏好 | 不同渠道话术模板 |
| 5分钟方案初稿 | 基于用户输入和附件资料，优先从方案池匹配并输出标准化方案结构 | 方案版本管理、模板库 |
| 成功案例引用 | 从成功案例池引用相似案例，说明适用条件 | 案例检索评分、案例标签体系 |
| 通用/个性化方案 | 区分成功案例池、通用预算池、专项个性化池 | 多模板、多字段池 |
| SKU组货建议 | 第一版先基于模板SPU和静态方案池输出组货策略，避免直接让AI从全量SKU中主观推荐 | 后续接入 SKU 图片、库存、价格检索 |
| 方案输出 | 在对话框内输出 Markdown 方案说明，并在右侧同步展示引用文件、Excel组货清单、PPT客户方案、PDF方案画册 | 接入PPT/PDF模板自动填充与在线预览 |
| 组货方案专家 | 根据预算、场景、品牌偏好、数量和交付约束输出三档组货建议；回答不了的问题自动记录为知识缺口 | 接入商品图、库存、价格和自动画册生成 |
| 知识库管理 | 支持上传组货逻辑、成功案例、预算池、个性化池、字段定义和模板资料；知识缺口进入待补队列 | 版本审核、灰度发布、语义层、知识缺口闭环 |
| MCP连接器 | 管理商品检索、库存图片、钉钉待办、钉钉文档、钉钉AI表格等连接器状态 | 更多业务系统连接器 |
| 触发器/定时任务 | 支持字段缺失、方案更新、库存待查、交付超时等提醒 | 周期巡检、自动复盘 |
| Skill编写管理 | 将预算分档、字段校验、品牌偏好判断等稳定规则封装为Skill | 一句话生成Skill、审核发布 |

## 5. 第一版明确不做

| 模块 | 第一版处理 |
|------|------------|
| 工作台首页 | 不做。首页搜索直接跳全屏对话工作台 |
| 能力市场 | 不做。只通过后台配置可用能力 |
| 需求提报/需求池 | 不做。后续单独作为需求闭环模块上线 |
| 完整能力市场 | 不做。只在全屏工作台保留必要能力入口 |
| 复杂创建智能体页面 | 不做。只保留方案中心主管与组货方案专家配置口径 |
| 大面积首页组件 | 不做。首页只保留搜索和少量快捷功能 |
| 数字人直播 | 不做 |
| 网络搜索开关 | 第一版可不暴露给用户，由后端策略控制 |

## 6. MVP 验收标准

| 验收项 | 标准 |
|--------|------|
| 首页 | 只有搜索/任务输入框、文件上传、基础快捷功能、少量示例，普通用户无后台入口，界面不出现MVP/试点字样 |
| 搜索跳转 | 首页输入问题后进入全屏对话工作台 |
| 全屏工作台 | 左历史、中对话、右能力上下文可用，页面不出现整体滚动 |
| 知识库 | 能上传知识资料并展示入库状态；组货专家无法回答的问题可沉淀为待补知识项 |
| 方案中心主管 | 能展示字段采集表，完成需求澄清、输出Markdown方案初稿、引用来源、Excel/PPT/PDF交付物状态，并继续追问优化 |
| 通用能力 | 通用问答、营销文案、文档总结、PPT提纲可作为快捷功能触发 |
| 钉钉能力 | 钉钉待办、钉钉文档、钉钉AI表格至少明确接口状态和降级方案 |
| 权限 | 普通用户不展示后台；未授权用户不可使用方案中心主管和受限知识库 |
| 审计 | 记录用户、会话、智能体、工具调用、权限拦截等基础日志 |

## 7. 后续迭代承接

| 阶段 | 方向 | 说明 |
|------|------|------|
| 二期 | 文件解析与知识库增强 | 支持附件在线预览定位、结构化抽取和字段回填 |
| 二期 | 方案中心能力增强 | 完善案例池、字段池、SKU 检索、PPT/PDF 生成 |
| 二期 | 需求池 | 上线需求提报、状态流转和AIBP协同 |
| 三期 | 能力市场 | 上线智能体、MCP、技能市场和安装/部署 |
| 三期 | 定时任务 | 提供任务触发器、日历、提醒和自动执行 |

## 8. 首发口径

> AI中台第一期 MVP 先让种子用户在一个入口里完成“问问题 - 生成方案 - 继续优化 - 调用基础协同能力”。先不做完整门户和需求闭环，避免范围发散，优先验证方案中心场景是否能跑通。
