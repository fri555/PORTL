### 需求4: 修改名称、移动位置

**需求：** 对有权限的文件夹支持修改名称、移动位置；对有权限的文件，支持移动位置，但不能超过本知识库

**当前现状：**

| 子项 | 状态 | 说明 |
|---|---|---|
| 文件重命名（列表+预览） | ✅ 已实现 | `beginRenameDoc` / `commitRenameDoc` |
| 文件夹重命名（双击改名） | ✅ 已实现 | `beginRenameFolder` / `commitRenameFolder` |

| 文件移动弹窗 | ✅ 已实现 | `fileActionModalOpen` |
| 文件移动不能跨空间 | ❌ **有 Bug** | 见下文 |

**Bug 详情 - `targetKbs` 跨空间：**
```ts
const targetKbs = computed(() => {
  const all = (Object.entries(knowledgeBases) as [SpaceKey, KnowledgeBaseItem[]][]).flatMap(([space, items]) =>
    items.filter(kb => kb.id !== selectedKbId.value).map(kb => ({ ...kb, space }))
  )
  // ...
})
```
此过滤条件**仅排除自身**，未过滤同空间，导致文件可以从公共空间移动到个人空间。

**解决方案：**
```diff
  items.filter(kb =>
+   space === activeSpace.value &&
    kb.id !== selectedKbId.value
  ).map(kb => ({ ...kb, space }))
```

## 2. 文件树左侧边栏

### 需求1: 文件树中增加权限标签

**需求：** 在文件树中对应知识库、文件也增加用户权限标签

**当前现状：** 未实现。文件树节点无权限标识。

- 右侧知识库列表（`displayedKnowledgeBases`）已有 `visibility` 字段展示（"全员可见"/"部门可见"/"仅自己可见"）
- 但左侧树中知识库行、文件夹行、文件行均无任何权限徽标

**解决方案：**
1. 新增辅助函数：
```ts
function getNodePermissionLabel(kbId?: string): { label: string; class: string } | null {
  if (!kbId) return null
  const kb = findKbById(kbId)
  if (!kb) return null
  const map: Record<string, { label: string; class: string }> = {
    '全员可见': { label: '公开', class: 'bg-green-50 text-green-600' },
    '部门可见': { label: '部门', class: 'bg-blue-50 text-blue-600' },
    '仅自己可见': { label: '私密', class: 'bg-zinc-50 text-zinc-500' },
    '方案中心可编辑': { label: '可编辑', class: 'bg-orange-50 text-orange-600' },
  }
  return map[kb.visibility] ?? null
}
```
2. 在树节点标签后渲染：
```vue
<span v-if="getNodePermissionLabel(kb.id)"
      class="ml-1.5 shrink-0 rounded px-1 py-0.5 text-[9px] font-medium leading-none"
      :class="getNodePermissionLabel(kb.id)!.class">
```

```ts
const treeDragState = reactive({
```

```vue
draggable="true"
@dragstart="onTreeDragStart(row)"

@drop.prevent="onTreeDrop(row)"
@dragend="onTreeDragEnd"
```

```ts
function onTreeDrop(target: TreeRow) {
```

## 5. 知识库问答右侧边栏

### 需求1: 回复时清屏

**需求：** 输入内容回复的时候，清空屏幕进行回答

**当前现状：** 未实现。`askKnowledgeBase()` 直接追加用户消息和 AI 回答，不涉及"清屏"效果。

**差距分析：**
- 需要"清空屏幕"的视觉效果——清除历史消息，保留正在处理的上下文
- 现有 `qaMessages` 是线性追加，没有状态管理
- 与主页对话界面的"刷新/清屏"行为不一致

**解决方案：**
```ts
function askKnowledgeBase() {
  const text = qaQuestion.value.trim()
  if (!text) return

  // 1. 添加用户消息（可见）
  const userMsg: QaMessage = { id: Date.now(), role: 'user', content: text }
  qaMessages.value.push(userMsg)

  // 2. 模拟清屏过渡（隐藏旧消息，300ms后只保留最新的问答对）
  setIsThinking(true)  // 新增 isThinking ref
  setTimeout(() => {
    // 只保留当前用户消息和待生成的 AI 回复
    qaMessages.value = [userMsg]
    // 添加"正在思考"占位
  }, 300)

  // 3. 模拟 AI 回复（延迟后）
  setTimeout(() => {
    const lastMsg = qaMessages.value[qaMessages.value.length - 1]
    if (lastMsg?.role === 'user') {
      qaMessages.value.push({
        id: Date.now() + 1,
        role: 'assistant',
        content: `已在「${selectedKb.value?.name ?? ''}」中完成检索。...`,
        citations: [...],
      })
    }
    setIsThinking(false)
  }, 1500)
}
```

**工作量：** ~15 行 | 难度：★★☆☆☆

### 需求5: 追问引用回复

**需求：** 追问引用 AI 回复时，回复卡片和豆包一样出现在输入框上方，不全部展示，只保留一行，多余的省略号

**当前现状：** 未实现。`citeQaMessage()` 仅仅是拼接文本：
```ts
qaQuestion.value = `引用上一条回答继续：${msg.content.slice(0, 60)}...`
```

**差距分析：**
- 没有在输入框上方渲染可关闭的引用卡片
- 直接修改输入框内容，用户看不到"引用了哪条消息"
- 和豆包/主页的追问交互模式不一致

**解决方案：**

1. 新增 `pendingQuoteRef` ref：
```ts
const pendingQuoteRef = ref<{ text: string; sourceMsgId: number } | null>(null)
```

2. `citeQaMessage()` 改为设置引用而非修改输入框：
```ts
function citeQaMessage(msg: QaMessage) {
  pendingQuoteRef.value = {
    text: msg.content.slice(0, 80),
    sourceMsgId: msg.id,
  }
}
```

3. 输入框上方渲染引用卡片：
```vue
<div v-if="pendingQuoteRef"
     class="mx-3 flex items-center gap-2 rounded-lg border border-zinc-200
            bg-zinc-50 px-3 py-1.5 text-xs">
  <Quote class="h-3 w-3 shrink-0 text-blue-500" />
  <span class="min-w-0 flex-1 truncate text-zinc-500">
    {{ pendingQuoteRef.text }}
  </span>
  <button @click="pendingQuoteRef = null"
          class="shrink-0 rounded p-0.5 text-zinc-400 hover:text-zinc-700">
    <X class="h-3 w-3" />
  </button>
</div>
```

4. 发送时自动将引用文本拼入消息：
```ts
function askKnowledgeBase() {
  const text = qaQuestion.value.trim()
  if (!text) return
  const finalText = pendingQuoteRef.value
    ? `[引用的上一条回答] ${pendingQuoteRef.value.text}\n——${text}`
    : text
  pendingQuoteRef.value = null
  // ...
}
```

**工作量：** ~20 行 | 难度：★★☆☆☆
