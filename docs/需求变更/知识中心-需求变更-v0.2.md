# 知识中心原型界面需求变更

---

## 文件导航界面（主view）

### 需求1: 点击分组卡片进入文件夹

**需求：** 点击根空间的分组卡片也同样支持进入空间文件夹视图

**现状：** 已实现。空间文件夹的卡片行（表格）包含「进入」按钮，点击调用 `openSpaceFolder(folder)` 进入文件夹视图。

### 需求2: 上传文件支持设置标签、密级

**需求：** 上传文件的弹窗中支持设置标签，给几个默认的选项，同时支持自定义标签

**现状：** 已实现。上传弹窗包含标签选择面板（8个预设标签：案例/模版/预算/分析/制度/SOP/团购/方案）、自定义标签输入、密级下拉（公开/内部/秘密/机密）。

### 需求3: 删除toast提示

**需求：** 删除文件、知识库时，提醒暂时存入回收站中，30天内可恢复

**现状：** 已实现。删除后调用 `showFileActionToast('已删除：xxx，可在回收站找回')`，并在回收站中展示剩余天数。

### 需求4: 修改名称、移动位置

**需求：** 对有权限的文件夹支持修改名称、移动位置；对有权限的文件，支持移动位置，但不能超过本知识库

**现状：** 部分实现。重命名支持（文件夹双击改名、文件改名按钮）；移动文件弹窗已实现（选择目标知识库，不能跨空间）；文件夹拖拽排序已实现（限同一父节点）。

## 文件树左侧边栏

### 需求1: 文件树中增加权限标签

**需求：** 在文件树中对应知识库、文件也增加用户权限标签

**现状：** 未实现。当前文件树节点无权限标识。

### 需求2: 文件树节点右键菜单

**需求：** 不同节点类型展示不同右键菜单项

**现状：** 已实现。5个分支：根空间空白处（新建空间文件夹/删除）、空间文件夹（新建知识库/设置/删除）、知识库（新建文件夹/设置/删除）、普通文件夹（新建文件夹/删除）、具体文件（预览/设置/删除）。

### 需求3: 文件树节点拖拽排序

**需求：** 文件树、中间主view知识库支持拖拽移动位置，文件夹移动不得超出知识库，知识库移动不得超出空间文件夹

**现状：** 已实现。空间文件夹、文件树中的知识库/文件/文件夹节点均支持拖拽排序，`GripVertical` 把手悬停显示，超出范围 Toast 提示。

## 文件上传

**现状：** 上传流程已实现前端模拟版本。

- 上传弹窗（标题、文件选择虚线框、文件列表）✅
- 格式/大小校验（PDF/DOCX/XLSX/PPTX/TXT/PNG/JPG）✅
- 标签选择（预设 + 自定义）✅
- 密级选择（公开/内部/秘密/机密）✅
- 上传任务浮层（最多4个、状态流转：uploading→pending→processing→done/reviewing）✅
- 文件状态徽标（上传中🔵/排队中🔵/处理中🔵/已完成🟢/待审核🟡/上传失败🔴）✅
- 待优化：魔数校验、SHA256去重秒传、解析失败重试为非模拟的真实 API 对接

## 回收站界面

### 需求1: 关闭按钮位置与自动关闭

**需求：** 关闭按钮应采用通用关闭交互，点击文件树节点时自动关闭回收站

**现状：** 已实现。移除 `ChevronLeft`，`X` 关闭按钮在右上角；切换空间/点击文件树节点时自动 `closeRecycleIfOpen()`。

### 需求2: 回收站功能

**需求：** 展示名称/类型/删除时间/剩余天数，支持恢复和彻底删除

**现状：** 已实现。列表展示文件类型图标、名称、删除时间、剩余天数标签；「恢复」和「彻底删除」按钮，30天后自动清理（`autoPurgeRecycle()`）。

## 权限管理界面

**现状：** 已满足 MVP 要求。知识库设置弹窗包含：
- 5级角色（OWNER/MANAGER/EDITOR/DOWNLOADER/READER）
- 成员列表（姓名/部门/角色下拉/加入时间）
- 添加成员搜索弹窗
- 四种预设规则（本部门读写/全公司阅读/指定部门/公开）
- 移除成员二次确认
- 审计日志筛选（日期/类型/用户）与展示

**待优化：** 文件级别权限设置、真实 API 对接。

## 知识库问答右侧边栏

### 需求1: 引导语

**需求：** 默认显示引导语，介绍小智

**现状：** 已实现。`qaMessages` 为空时展示引导卡片（「你好，我是小智」）+ 三个预设问题按钮。

### 需求2: 快捷功能

**需求：** 四个快捷功能卡片（智能问答/知识检索/图谱分析/数据洞察），点击后自动把 prompt 填入输入框

**现状：** 已实现。四个模式按钮：智能问答（answer）、知识检索（search）、图谱分析（graph）、数据洞察（insight），点击切换模式并显示对应引导文本。

### 需求3: 回复时思维链

**需求：** 小智回复问题要有思维链过程，和主页中对话界面保持一致

**现状：** 未实现。当前回复为纯文本 + 引用标签，无思维链展示组件。

### 需求4: 引用溯源文件卡片

**需求：** 引用文件来源使用卡片的形式，和主页中对话界面保持一致

**现状：** 未实现。当前引用以文本列表 + 引用序号展示，非卡片样式。
