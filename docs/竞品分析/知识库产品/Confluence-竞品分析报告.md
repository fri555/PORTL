## 1. 产品概述

### 1.1 Confluence 是什么

**Confluence** 是澳大利亚 Atlassian 公司推出的企业级 Wiki 知识管理平台，是 **全球企业 Wiki 事实标准**。自 2004 年发布以来，Confluence 建立了最成熟的企业级内容治理体系——页面树、空间隔离、版本管理、审批工作流、插件生态。与 Jira 的深度集成使其成为研发团队的首选知识管理工具。

### 1.2 目标用户

- 已使用 Jira 的中大型研发团队
- 需要复杂内容治理体系的传统企业
- 多团队/多项目的大型组织

### 1.3 公司概况

| 项目 | 内容 |
|------|------|
| **开发商** | Atlassian（澳大利亚） |
| **用户规模** | 全球数百万企业用户 |
| **部署方式** | Cloud + Data Center（私有化） |
| **AI 引擎** | Atlassian Intelligence |
| **核心生态** | Jira + Bitbucket + Confluence |

## 4. SWOT 分析

| 优势 | 劣势 |
|------|------|
| S1. 内容治理全球最成熟 | W1. AI能力弱，落后AI原生代差 |
| S2. Jira生态深度绑定 | W2. 编辑器加载慢，体验重 |
| S3. 版本管理/审批链完善 | W3. 成本高（按人头+服务器+插件） |
| S4. 插件生态丰富 | W4. 国内访问体验差 |
| S5. Data Center私有化 | W5. 用户流失趋势明显 |

| 机会 | 威胁 |
|------|------|
| O1. 存量用户短期不会全量迁移 | T1. 大量用户正在寻找替代方案 |
| O2. 企业级治理需求长期存在 | T2. 飞书/钉钉AI能力代际领先 |
| O3. 与Jira的集成是迁移壁垒 | T3. 本土产品性价比碾压 |

## 5. 战略建议

Confluence 用户正在大量寻找替代方案——高成本、重体验、AI能力弱是三大痛点。**这是天马智擎的重要市场窗口**。

| 优先级 | 建议 | 说明 |
|--------|------|------|
| 🔴 P0 | **核心替代卖点：更低的成本 + 更轻的体验 + 更现代的AI能力** | 这是Confluence用户最关注的三点 |
| 🟡 P1 | **Confluence页面树参考** | 成熟的页面层级管理值得借鉴 |
| 🟡 P1 | **版本管理参考** | 对比/回滚/审批链参考设计 |
| 🟢 P2 | **Data Center客户定向转化** | 私有化部署需求是转化契机 |
